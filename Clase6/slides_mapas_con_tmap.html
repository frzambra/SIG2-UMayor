5. Configuraci√≥n del color

  • Como defino la leyenda de colores no es algo trivial. Muchos problemas de interpretaci√≥n de resultados se generan por una mala visualizaci√≥n.

  • La configuraci√≥n predeterminada de {tmap} usa pretty breaks.

Los argumentos:
- breaks permite manualmente ajustar los breaks
- n define el num√©rp de bin en el cual la variable es categorizada
- palette define el esquema de colores

5. Configuraci√≥n del color

5. Configuraci√≥n del color

  • Podemos modificar la escala de colores con el argumento breaks

  • Adem√°s de poder cambiar los breaks de forma manual {tmap} permite usar algor√≠tmos para definir la escala.

Los seis m√°s utilizados algor√≠tmos son:
- style = "pretty", la configuraci√≥n predeterminada, redondea los breaks en n√∫meros enteros cuando sea posible y los espacia uniformemente;
- style = "equal" divide los valores de entrada en contenedores de igual rango y es apropiado para variables con una distribuci√≥n uniforme ;
- style = "quantile" asegura que el mismo n√∫mero de observaciones caiga en cada categor√≠a ;
- style = "jenks" identifica grupos de valores similares en los datos y maximiza las diferencias entre categor√≠as;
- style = "cont" (y ‚Äúorder‚Äù) presentan una gran cantidad de colores sobre campos de color continuos y son particularmente adecuados para r√°steres continuos (‚Äúorder‚Äù puede ayudar a visualizar distribuciones sesgadas);
- style = "cat" fue dise√±ado para representar valores categ√≥ricos y asegura que cada categor√≠a reciba un color √∫nico.
